# アクセシビリティ手動確認項目（VoiceOver）

自動テスト（`app/client/e2e/accessibility.pw.ts`・CI の `accessibility` ジョブ）はキーボード操作・ARIA属性・axe の serious/critical 0 を検証する。実際の読み上げ内容と操作感は自動化できないため、以下を macOS VoiceOver（Cmd+F5）で手動確認する。関連Issue: #185 / #210 / #211。

## 画面遷移の通知（#210）

- [ ] サイドバーの各項目（例: 進捗）を押すと、新しい画面名（「進捗に移動しました。」等）が読み上げられる
- [ ] ブラウザの「戻る」「進む」でも同様に新しい画面名が読み上げられる
- [ ] 遷移後に VO カーソル/フォーカスが本文（main）側にあり、そこから新画面の内容を順に読める
- [ ] ブラウザのタブ名・履歴一覧が「画面名 — solo-eikaiwa」で画面ごとに区別できる
- [ ] UI言語を EN / 日本語 で切り替えると、タブ名と以後の遷移読み上げが選んだ言語になる

## 設定画面のタブ（#185 / #211）

- [ ] 設定を開くと「表示」タブが選択済みで、文字サイズ・言語へすぐ到達できる
- [ ] タブ列で VoiceOver が「タブ」「選択中」「◯/4」を案内する
- [ ] Tab キーではタブ列全体が1ストップ（選択中タブのみ）で、次の Tab でパネル内容へ進む
- [ ] ← / → / Home / End でタブ間を移動でき、移動と同時に選択・表示が切り替わる（端では折り返す）
- [ ] 各タブとパネルが対応づけて案内される（パネルに入るとタブ名がラベルとして読まれる）
- [ ] 設定の読み込みエラー・読み込み中の案内が、選択中のパネル内で読み上げられる
- [ ] `#/settings?tab=keys` を直接開くと APIキータブが選択された状態で開く

## whisper セットアップバナーのモデル選択（#211）

- [ ] モデル選択で VoiceOver が「ラジオボタン」「選択中」を案内する
- [ ] ↑↓←→ どの矢印キーでも選択が切り替わり、読み上げが追従する
- [ ] Tab キーでは選択中のラジオ1つだけに止まり、他の操作（開始ボタン等）へ進める
