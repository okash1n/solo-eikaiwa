# 練習日・発話指標の定義

進捗画面、月次振り返り、レベル提案で使う母集団と式をここで固定する。率や平均は、日別に丸めた値を再平均せず、対象期間の分子・分母を合算してから計算する。

## 練習日

ローカル日付のセッションログに `user_utterance`、`stt_result`、完了した `round_end`、完了した `block_end` のいずれかがある日、または同日に XP イベントがある日を練習日とする。`session_start`、`session_end`、閲覧・開始だけのイベント、`aborted: true` の終了は数えない。

日付はイベント書き込み時のローカル日付で区切る。日付を跨いだ練習は、意味のあるイベントが発生した各日に記録される。既存ログは削除せず、表示時に同じ定義で再評価するため、過去の起動だけの日は表示から外れる。

例外: レベル提案（ステージ境界の昇格判定・`progress-store.ts`）の練習日だけは、この定義ではなく**ブロック完了XP（`kind = 'block'` の XP イベント）がある日**を直近14日窓で数える。発話や例文の自己評価だけの日は昇格判定の練習日に含めない。進捗画面・月次振り返りの練習日は本節の定義のままとする。

## 発話指標

母集団は、対象期間の `stt_result` のうち `metrics.words` と `metrics.speechMs` を持つイベントとする。旧形式などこの2値がないイベントは全発話指標から除外する。ポーズ情報がないイベントはポーズ率の分子・分母から、反復率がないイベントは反復率の分子・分母からだけ除外する。0件または指標固有の分母が0の場合、その指標は0とする。

- 発話数: 対象 `stt_result` の件数
- 発話時間（秒）: `round(Σ speechMs / 1000)`
- 構音速度（WPM）: `round1(Σ words / (Σ speechMs / 60000))`
- ポーズ率: `round3(Σ pauseMs / Σ totalMs)`
- 反復率: `round3(Σ (utterance repetitionRatio × utterance words) / Σ repetitionRatio観測済みwords)`

週次比較はローカル当日を含む直近7日と、その直前7日を別々に上式で集計する。月次振り返りはローカル当日を含む直近30日を同じ式で集計する。

## 完了率・自己評価率

- ブロック完了率: `completed block attempts / eligible block attempts`。明示完了・明示中断・旧形式の試行を母集団とし、通信やDB障害で結果が未確定の試行は除外する。0件は `null` とする。
- SRS Good率: `XP amount = 2 の srs-grade events / 全 srs-grade events`。0件は `null` とする。

## 振り返り範囲

通し練習を開始するたびにクライアントが練習IDを生成する。会話の各発話と振り返り要求を同じIDへ結び付け、日付ファイルを跨いで対象IDだけをtimestamp順に取得する。別タブ、別の通し練習、単独の自由会話は混在させない。
