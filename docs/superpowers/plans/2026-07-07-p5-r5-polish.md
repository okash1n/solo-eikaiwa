# P5 + R5 仕上げ（小粒の非対称解消・リファクタリング） Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** UX一貫性レビュー（spec §3 P5・§5 R5）と R4+P4 最終レビュー繰越の小粒項目を消化し、画面間の機能非対称・外殻文言のハードコード・テストのボイラープレートを解消して v0.8.0 を仕上げる。

**Architecture:** 既存の確立イディオムに全面的に乗る。クライアントの「訳・解説」ボタンは `useExplain(fetcher)` フックの1インスタンス子コンポーネント方式（AeItemView と同型）に統一し、fetcher は既存 api バレルの `fetchTalkExplanation` / `fetchFixExplanation` / `fetchUtteranceTranslation` / `fetchSentenceExplanation` を流用する。新規エンドポイントは作らない。純関数は独立モジュールへ抽出してユニットテストする。新規 UI 文言は i18n 辞書（EN/JA 両方）へ追加する。

**Tech Stack:** TypeScript / React 18 / Bun test / Vite。サーバは Bun の `fetch` ハンドラ + 機能別ルータモジュール。

## Global Constraints

これらは spec と本プロジェクトの確立規約から来る、全タスク共通の制約。各タスクの要件はこの節を暗黙に含む。

- **研究制約（情報的フィードバックのみ）**: 訳・解説はユーザーの明示操作（ボタン押下）でのみ表示する。自動表示・先読み表示はしない。プレースメント（レベル測定）は測定用途なので、お題「本文」の訳はユーザー起点のボタンに限り、スコアリング支援にはしない。
- **i18n 文言のハードコード禁止**: P4 完了後のため、新規 UI 文言はすべて `app/client/src/i18n.ts` の `STR.en` と `STR.ja` の両方に追加する。named 型（例: `LibraryStrings`）にキーを足し、両言語のオブジェクトに同じキーを埋める。
- **既存イディオムに従う**: ルータは機能別モジュール + `[method, path, handler]` テーブル（`app/server/routes/*.ts`）。api はドメイン別モジュール + `app/client/src/api/index.ts` バレル。解説ボタンは `useExplain`（`app/client/src/useExplain.ts`）。リスト行TTSは `usePlayRow`。マウント時ロードは `useLoad`。
- **挙動契約**: 本計画の UI 改善（項目5〜9・11・3）はいずれも小さな意図的改善で、未設定・既定の利用者体験は v0.8.0 と同一に保つ（新しいボタンは押さなければ何も起きない）。項目10・12・13 は挙動不変の純リファクタ/テスト追加。
- **検証ゲート（各タスク末尾で必ず実行）**:
  - `cd app && bun test`
  - `cd app && bun run typecheck`
  - `cd app/client && bun run build`

## Task 1: クライアント解説系の非対称解消（P5-1・P5-2・繰越5）

「訳・解説」ボタンの有無の画面間非対称を、`useExplain` 1インスタンス子コンポーネント方式に揃えて解消する。3サブ変更を1コミットにまとめる。

- **P5-1**: ライブラリ（モデルトーク一覧）に訳解説ボタンを追加（`fetchTalkExplanation` 流用 — シャドーイングと同じ talk-explain）。
- **P5-2**: 一覧タブの「マイチャンク」カードに「もっと詳しく」を追加。プロンプトは**新規作成せず `fetchFixExplanation` を流用**する。判断根拠: マイチャンクの1件は `{ promptText=学習者の元の言い方, en=自然な言い方, note=理由 }` という構造で、これは fix-explain の入力 `{ original, better, note }` にそのまま一致する。fix-explain はキャッシュなし・ボタン起点のオンデマンド生成で、一過性データのチャンク解説に適する。新エンドポイント・新プロンプトは不要。
- **繰越5**: 練習タブ（PracticeTab）の例文解説を、旧来のセンチネル文字列 state（`"loading"` とエラー文言を本文に混ぜる方式）から `useExplain` に統一する。これで「エラー後に再試行できない」問題と「言語切替後にエラー文言が旧言語のまま残る」問題（エラー文言を state に保存していたため）が同時に直る。

- [ ] **Step 2: LibraryScreen をエントリ子コンポーネントに分離して訳解説を追加**

`app/client/src/screens/LibraryScreen.tsx` を以下に置き換える。`usePlayRow` は共有（1行のみ再生）なので親に残し、`useExplain` はエントリ単位で `LibraryEntry` に持たせる（エントリごとに独立した解説状態）。

```tsx
import { fetchModelTalkLibrary, fetchTalkExplanation, type ModelTalkEntry } from "../api";
import { STR, type Lang } from "../i18n";
import { useLoad } from "../useLoad";
import { usePlayRow } from "../usePlayRow";
import { useExplain } from "../useExplain";
import { Banner } from "../ui/Banner";
import { Button } from "../ui/Button";
import { Card } from "../ui/Card";

/** 生成済みモデルトークの一覧（情報表示のみ）。本文確認・再再生・訳解説ができる。 */
export function LibraryScreen({ lang }: { lang: Lang }) {
  const t = STR[lang].library;
  const { state, reload } = useLoad(fetchModelTalkLibrary);
  const row = usePlayRow<number>();

  return (
    <div>
      <h3>{t.title}</h3>
      {state.status === "loading" && <p className="text-muted">{t.loading}</p>}
      {state.status === "error" && (
        <Banner kind="error" action={<Button onClick={reload}>{t.retry}</Button>}>{state.error}</Banner>
      )}
      {state.status === "ready" && state.data.length === 0 && (
        <p className="text-muted">{t.empty}</p>
      )}
      {state.status === "ready" &&
        state.data.map((e) => (
          <LibraryEntry key={e.id} entry={e} lang={lang} row={row} />
        ))}
      {state.status === "ready" && row.error && <Banner kind="error">{row.error}</Banner>}
    </div>
  );
}

/** 1エントリ: 再生（共有 row）＋本文折りたたみ＋訳解説（talk-explain 流用・エントリ単位の useExplain）。 */
function LibraryEntry({ entry, lang, row }: {
  entry: ModelTalkEntry; lang: Lang; row: ReturnType<typeof usePlayRow<number>>;
}) {
  const t = STR[lang].library;
  const explainer = useExplain(() => fetchTalkExplanation(entry.text));
  return (
    <Card
      header={
        <>
          <Button
            variant="ghost"
            onClick={() => row.play(entry.id, entry.text)}
            disabled={row.playingKey !== null}
            ariaLabel={t.playAria(entry.topicTitle || entry.topicId)}
          >
            {row.playingKey === entry.id ? t.playing : "▶"}
          </Button>{" "}
          {entry.topicTitle || entry.topicId}{" "}
          <span className="text-sm text-muted">{entry.createdAt.slice(0, 10)}</span>
        </>
      }
    >
      <details>
        <summary className="text-muted">{t.transcript}</summary>
        <p className="reading-text">{entry.text}</p>
      </details>
      {explainer.state.status === "idle" && (
        <Button variant="ghost" onClick={explainer.request}>{t.explainMore}</Button>
      )}
      {explainer.state.status === "loading" && <p className="text-sm text-muted">{t.explainLoading}</p>}
      {explainer.state.status === "error" && (
        <p className="text-sm text-muted">{t.explainError}<Button variant="ghost" onClick={explainer.request}>{t.retry}</Button></p>
      )}
      {explainer.state.status === "done" && <p className="sentence-explain text-sm">{explainer.state.text}</p>}
    </Card>
  );
}
```

/** ブロックの表示タイトルを言語別に組み立てる。titleKey の無い旧キャッシュ（デプロイ当日など）は従来の JA title をそのまま使う。

function PlacementPrompt({ text, lang }: { text: string; lang: Lang }) {
