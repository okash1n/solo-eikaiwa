# Changelog

このプロジェクトの特筆すべき変更を記録します。形式は [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/) に、バージョニングは [Semantic Versioning](https://semver.org/lang/ja/) に従います。

## [0.29.3] - 2026-07-17

### Fixed

- 暗記例文・マイフレーズの評価送信が失敗した後に別の評価ボタンを押したとき、初回の評価が無言で再送されて選び直しが記録されない問題を修正した。選び直した評価がそのまま記録されるようになり、初回の評価がすでにサーバへ記録済みだった場合はその旨を表示して次のカードへ進む

### Added

- アプリ内から開けるプライバシーポリシーと公開サポートページを追加した

### Changed

## [0.29.1] - 2026-07-12

### Changed

- GitHub Releasesのデスクトップ版をDeveloper IDで署名し、Apple公証とstapleを完了した状態で公開する標準経路へ切り替えた。v0.29.0から初めて更新する際は、署名要件の変更によりマイク許可を一度だけ再確認される場合がある
- 「このアプリについて」画面への遷移を廃止し、サイドバー最下部からGitHubリポジトリと公式サイトを2つのアイコンで直接開けるようにした。BTAJPの著作権とMITライセンス表記も同じフッターへ追加した
- APIキーと認証方式を専用タブへ集約し、Claude・Codex・OpenAI公式・OpenAI互換LLM/TTSの設定有無と保存元を一覧で確認できるようにした。公式OpenAIは固定URLと専用キー、互換接続は独自URLと接続先別キーとして分離し、利用可能な接続だけを用途へ選択できる
- TTSの暗黙的な自動選択を廃止し、macOS say・OpenAI公式・OpenAI互換を明示選択する方式に変更した。HTTP合成失敗時は別エンジンへ黙って切り替えず、旧公式URL設定は公式OpenAIへ自動移行する

### Fixed

- デスクトップリリースのdmg生成をFinder操作に依存しない方式へ変更し、途中失敗でTauriの生成スクリプトが残っても次の検証を壊さず再実行できるようにした
- remoteのOpenAI互換接続を、接続先へ承認済みのAPIキーが無い状態で用途へ新規割当できないようにした。モデル一覧の取得にも同じ承認済みキーを使用する

## [0.29.0] - 2026-07-11

### Added

- WAL稼働中でもトランザクション整合性を保つSQLite backup/restore CLIを追加した。snapshotはmanifest・SHA-256・schema fingerprint・integrity checkで検証し、復元前の現DBを自動退避して、切替失敗時は元データへ戻す
- 暗記例文とマイフレーズを英文・日本語・番号・カテゴリで検索できるようにし、390件の例文一覧は40件ずつ表示するようにした。次回復習日と学習段階も読みやすく表示する
- 4/3/2と振り返りで自動収集したマイフレーズについて、実際に保存できた表現・追加なし・保存失敗を区別して表示し、保存後は一覧を直接開けるようにした

### Changed

- デスクトップ版の音声認識エンジンを、SHA-256で検証する固定sourceからstaticビルドする方式に変更した。ReleaseにはSBOM・第三者ライセンス・native依存manifest・依存監査結果・checksumを添付し、配布内容を確認できるようにした
- 4/3/2の文字起こし表示の話者ラベルを画面言語に合わせ、使われなくなった練習ブロックと設定文言を整理した
- READMEとLPで、デスクトップ版の入手・自動更新、ソース常駐版の日常更新、Keychainと`app/.env`のキー優先順位を実装に合わせて案内するようにした
- 教材生成CLIのお題・シナリオ生成を品質ゲート付きの `--fill-coverage` / `*-target` へ統一し、旧 `topics` / `scenarios` / `topics-band` は移行案内を出して書き込まないようにした
- TTSの生成キャッシュを接続先・エンジン・モデル・voice・生成形式ごとに分離した。旧方式のキャッシュは削除せず参照対象外になるため、更新後の初回再生だけ再合成され、クラウドTTSではその1回分の課金が発生しうる
- 開発・常駐化・デスクトップビルド・リリース時のBun依存解決をfrozen lockfileへ統一し、Bun/Tauri CLIの検証済みexact版と異なる環境では処理を開始しないようにした
- 教材生成・検証スクリプトも通常のTypeScript型検査へ含め、サーバ変更によるCLI破損を事前検出するようにした
- PR・ローカル・リリースで同じ検証スクリプトを使い、Bun/TypeScript/client buildに加えてShellCheck・教材品質・desktopのRust test/clippyを継続確認するようにした
- Bun/Cargo依存監査を週次・手動・リリース前に追加し、脆弱性検出と監査サービス障害を区別して報告するようにした

- ホームのレベル測定案内を初回・月次とも同じ予約枠に表示し、取得失敗時も位置を保ったまま再試行できるようにした。マイフレーズの未学習・復習済み絞り込み、カードの小サイズ文字階層、画面言語の即時反映も整えた
- ライブラリが空のときの教材読込、メニューキャッシュの破損判定、診断ログのローカルパス伏せ字、デスクトップ更新の0バイト進捗処理を改善した

- 強化セッションとクイックドリルで、教材の準備完了前にタイマーや進捗記録が始まらないようにした。音読・再生・録音・振り返りなどの実施前はブロックを完了できず、途中で離れたブロックは中断として扱われてXPを付与しない
- 設定画面の接続・用途割当・TTSを別々の保存単位にし、他タブで編集中の値を別の保存操作で確定したり、遅い応答で入力を戻したりしないようにした。未保存状態、保存対象、認証モードの確定操作を画面で明示し、保存中は設定操作を保護する。APIキーの保存・削除には進行表示を付け、削除は2段階確認に統一した。TTSの既定化も入力欄で確認してから保存する方式にした
- Whisperモデルの初回セットアップを後回しにしても、常設の状態案内や録音開始時の案内からダウンロード画面へ戻れるようにした。途中のダウンロードも同じ画面で進捗・再開状態を確認できる
- 自由会話・ロールプレイ・4/3/2・レベル測定では、録音を始める前に音声認識モデルとLLMの準備を確認するようにした。不足時は録音や最後の採点まで進めず、その場でモデルのダウンロードまたはLLM設定を開ける
- 空の文字起こしを発話量ではなく技術的な失敗として扱い、自由会話・レベル測定・4/3/2でその場から録音し直せるようにした。4/3/2の能力シグナルにも含めない
- 日本語ヒント（お題の骨組み・代替表示を含む）と4/3/2のモデルトークのスクリプトを設定やフェーズ遷移だけで開示しないようにした。各教材で表示・再生を押した後だけ確認でき、次の教材では再び閉じた状態から始まる。モデルトークの事前準備設定とスクリプトの呼称も日英UIで統一した
- マイチャンクの削除を可逆な非表示へ置き換えた。非表示中は一覧と復習対象から外れるが学習データは保持され、非表示一覧からいつでも復元できる
- OpenAI互換LLMの接続先をこのMac・LAN・remote・無効URLに分類し、設定中と用途ごとの実効表示にoriginを追加した。About・README・LPも、ローカル保存と選択providerへの外部送信、認証・課金先を区別する説明へ訂正した
- 多聴の聴取記録をクライアント生成IDで冪等化し、保存失敗時は聴取完了と感想欄を維持したまま中立な案内から再試行できるようにした。応答消失後の再送でも週カウントを二重加算しない
- 4/3/2の準備チャンクやモデルトークの生成結果が空だった場合は恒久キャッシュせず、既存の空キャッシュも次回アクセス時に再生成して自己修復するようにした
- 例文の「1日の新規」を初回採点日にもとづく日次上限にし、明日の復習予定を例文と収集チャンクの合計「件数」で表示するようにした。モデルトークの日付と月次レビュー生成可否も閲覧環境のローカル暦日に統一した
- 教材候補の題名・hintに改行や引用符が混じった場合を拒否し、全Markdownをserialize後に再読して完全一致したときだけ一括書き込みするようにした。topic-assets生成は一過性のrunner例外を3回まで再試行する
- Whisperモデルのセットアップ表示を閉じた後に遅延した進捗確認がtimerや完了通知を復活させる競合と、キャンセル結果を古い進捗で一時的に巻き戻す競合を修正した
- LaunchAgentとデスクトップ同梱サーバの診断ログを1ファイル5 MiB・3世代でrotationし、長期利用時の合計容量を制限した。rotation中もサーバを継続し、APIキー・認証header・発話本文を示す行は保存前にredactする
- 起動時のmacOS Keychain読取を合計5秒で打ち切って子プロセスを終了し、環境変数の鍵で継続するようにした。LaunchAgentのログインシェルPATH取得も3秒で打ち切るため、Keychainやシェル初期化が応答しなくても無応答デーモンとして残らない
- macOS `say` の音声をAAC/M4Aで直接生成し、ffmpegを同梱しないデスクトップ版でもAPIキーなしの読み上げが動くようにした
- TTS通信とmacOS `say`を合計60秒で中断し、要求キャンセル時も通信・子プロセス・一時ファイルを残さないようにした。`TTS_API_KEY`だけを設定した場合もヘルス表示へ反映する
- TTSキャッシュを一時ファイルから原子的に公開し、書き込み失敗や強制終了で部分音声が恒久キャッシュされないようにした
- 音声設定・TTSキー変更時にタブ内の通常音声とモデルトークを即時無効化し、それぞれ64件・16件のLRU上限でBlobを回収するようにした
- LLM呼び出しの180秒上限を実処理の中断まで伝え、代替経路へ切り替わっても合計待ち時間が延びないようにした
- OpenAI互換・Codexの一時会話履歴をturn数・推定token数・セッション数・未使用時間で制限し、古い履歴を自動回収するようにした
- レベル測定と4/3/2の録音を制限時間で自動停止・文字起こしし、タブのタイマー抑制に左右されない実録音時間を記録するようにした。手動停止との競合でも二重送信しない
- マイク許可待ちで画面を離れた場合に、あとから許可されたstreamを即座に解放するようにした。録音開始の連打も1要求に直列化し、準備中であることを日英UIへ表示する
- TTSの取得待ち中に停止・採点・画面移動した場合、取得完了後に古い音声が鳴り始めないようにした。古い再生の遅延失敗も現在の音声を停止不能にしない
- 録音ファイル名を要求ごとに一意化し、同一時刻の並列文字起こしによる上書きを防いだ。ローカルSTTへ同時実行・待機・timeout・cancelの上限を追加した
- 週次・月次の構音速度・ポーズ率・反復率を日別率の再平均ではなく、発話ごとの共通分子・分母から加重集計するようにした
- アプリを開いただけの日を練習日から除外し、発話・明示完了・XP記録がある日だけを表示するようにした
- セッション振り返りを練習ごとのIDで抽出し、同日の別練習を混ぜず、日付境界を跨いだ発話も順序どおり扱うようにした
- レベル提案の承認・却下を表示中の種別と移動先に束縛し、内容が変わった場合は適用せず最新状態へ更新するようにした
- 例文・収集チャンクの自己評価を回答IDで冪等化し、SRS状態とXPを同一transactionで更新するようにした。通信再試行や更新途中の失敗でも二重加算・片側だけの記録を残さない
- セッションの遅延応答・逆順応答・連打で架空のブロック開始や別ブロックの試行完了が記録されないようにした
- ブロック完了XPの記録失敗を中立的に表示して再試行できるようにし、技術的に未確定な試行を完了率・中断数に基づく調整提案から除外した
- 既存の学習DBに必須列・型・indexの不整合がある場合、機能利用中のSQLエラーになる前に起動時点で検出するようにした。不完全なDBには書き込まず、対象DB・期待値と実値・非破壊のバックアップ手順を表示する
- Whisperモデルのダウンロード応答が登録サイズを超えた場合、Content-Lengthの有無や正確さに依存せず受信を中止し、破損した一時ファイルを破棄するようにした
- Keychainに項目がないenv由来APIキーへ削除APIを呼んでも、稼働中のenv値と検出状態を失わないようにした
- ClaudeのAPIキーモード中に鍵が無くなった場合、サブスクリプション認証へ無言で切り替えず、設定画面とAPI応答で不整合を通知するようにした

### Security

- デスクトップ更新の公開前に、生成した更新アーカイブと署名を実行時と同じ検証器で照合し、直前リリースからの署名鍵連続性を強制するようにした。鍵ローテーションは旧鍵で署名する橋渡しリリースを明示指定した場合だけ実行できる
- デスクトップ版が既存サーバへ接続する条件を、アプリ版・handshake protocol・build・正規化データ領域・永続instanceの完全一致へ強化した。開発サーバ、更新前の旧sidecar、別データ領域、偽healthには接続せず、旧sidecarと一般的なポート占有を画面で区別する
- Bun/Viteが自動読込する`.env.local`や環境別`.env`を全階層でGit管理対象外にし、秘密情報の誤コミットを防止した
- ローカルAPIで許可するHost・Originをloopback、Caddy、Viteの正規経路に限定し、cross-siteリクエストと認証なしの非loopback待受を処理前に拒否するようにした
- JSONを`application/json`系かつ64 KiB以下のobjectに限定し、構造量と各フィールドを実行時検証するようにした。STT音声は対応形式かつ24 MiB以下、TTSテキストは8,000文字以下に制限した
- Keychain値をプロセス環境へ展開せず、すべてのサーバ子プロセスを固定allowlist環境で起動するようにした。Claude/Codexのサブスクリプション認証では親環境のAPIキーを除去し、APIキーモードでは対象providerの鍵だけを渡す
- OpenAI互換LLM・TTSの鍵を正規化済みoriginへ束縛し、別origin・非loopback HTTPへ送らないようにした。認証付き通信はredirectを追従せず、userinfo・非HTTP(S) scheme・曖昧なquery/fragmentを接続設定で拒否する

### Added（デスクトップ配布・更新）

- **デスクトップアプリの自動アップデート（半自動）**: 起動時に新しいバージョンを自動チェックし、あればダイアログでお知らせするようになった。「更新する」を押すとダウンロード・適用・自動再起動まで完了する（「今回はしない」を選べば何も起きず、次回起動時にまた1回だけ案内される。強制更新はしない）。アプリメニューに「アップデートを確認…」も追加し、いつでも手動で確認できる。更新パッケージは署名検証（minisign）に合格したものだけが適用される

## [0.1.0] - 2026-07-06

最初のリリース。検証済みの第二言語習得（SLA）研究に基づく、ローカルファーストな英会話独習アプリ。

### Added

- **音声ループ**: ブラウザ録音 → whisper.cpp（ローカルSTT）→ Claude（会話相手・コーチ、Agent SDK 経由でサブスクリプション認証）→ OpenAI TTS（未設定時は macOS `say` にフォールバック）
- **クイックドリル（5〜10分・日次のデフォルト導線）**: 音読ウォームアップ / 4/3/2ミニ / ロールプレイ / シャドーイングの単品ドリル
- **強化セッション（60分 / 30分）**: 音読ウォームアップ → 4/3/2 流暢性トレーニング → 実務ロールプレイ → シャドーイング → 振り返り の通し構成
- **4/3/2 流暢性トレーニング**: 準備フェーズ（お題別の表現チャンク・話の骨組み・モデルトーク聴取）→ 時間圧ラウンド（2分→1.5分→1分、メニュー設定駆動）→ ラウンド間の明示的フィードバック（エラーログ＋日本語解説）
- **自由会話モード**（AIとの英語フリートーク）
- **コンテンツローテーション**: お題（`content/topics/`）とシナリオ（`content/scenarios/`）を frontmatter 付き Markdown で管理、least-recently-used で自動選択
- **チャンク音声再生**: 各表現の🔊ボタン（押下時のみ生成、以降はキャッシュ即再生）
- **モデルトーク**: 準備画面表示時の先読み生成・段階表示・本文の常時表示
- **ライブラリ**: 生成したモデルトークを SQLite（`bun:sqlite`）に記録し、一覧・本文確認・再再生
- **練習日カレンダー**（直近8週・情報表示のみ）と **if-then アンカー**（習慣化研究に基づく1行プラン）
- **ローカルHTTPS**: Caddy 用サイト定義（`https://learn-english`）
- **セッションログ**: 全イベントを JSONL に記録（将来のメトリクス分析用・ローカル専用）
- **ドキュメント**: 設計スペック、実装計画、ディープリサーチレポート3本（流暢性・語彙/チャンク・継続/習慣化 — 主要クレームは原典まで検証済み）

### Security / Privacy

- サーバは 127.0.0.1 バインドのみ（外部非公開）
- 学習データ（録音・トランスクリプト・進捗・DB・キャッシュ）は `data/` のローカルファイルで、リポジトリにコミットされない
- 音声データはマシン外に出ない（外部送信はテキストのみ: Claude への発話テキスト、OpenAI TTS への応答テキスト）
